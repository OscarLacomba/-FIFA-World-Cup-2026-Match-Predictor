## Features

- **Match Prediction** — predicts Home Win / Draw / Away Win with confidence scores
- **23 engineered features**:
  - Elo ratings (pre-tournament snapshot for all 48 qualified teams)
  - Recent form — points/goals averages over last 5 and 10 matches
  - Win streaks
  - Days since last match (fatigue/inactivity)
  - Head-to-head historical record between each pair of teams
  - Home/away differential features
- **Class-balanced training** — `class_weight='balanced'` so the model doesn't just predict "Home Win" every time
- **Monte Carlo Simulation** — simulates the full tournament 1,000 times with precomputed pairwise probabilities
- **Web App** — 4-tab Gradio interface: Predictor, Matches, My Prediction, Leaderboard
- **SQLite Database** — stores users, matches and predictions persistently
- **AI Analysis** — Claude API generates natural language match explanations (with statistical fallback)
- **Leaderboard** — real-time ranking of users vs AI model

## Installation

### Run in Google Colab (recommended)

1. Open `WC2026_Predictor_v3.ipynb` in [Google Colab](https://colab.research.google.com/)
2. Run all cells in order (the notebook is self-contained)
3. The Gradio app will generate a public share link automatically

### Run locally

```bash
git clone https://github.com/YOUR_USERNAME/FIFA-World-Cup-2026-Match-Predictor.git
cd FIFA-World-Cup-2026-Match-Predictor

pip install -r requirements.txt

# After generating wc2026_model.pkl, wc2026_scaler.pkl, wc2026_features.pkl and wc2026.db
python app.py
```

## Usage

### 1. Train the model

Run cells 0–4 in the notebook. This will:
- Download the datasets via `kagglehub`
- Engineer 23 features (Elo, recent form, streaks, head-to-head, rest days)
- Train Logistic Regression, Random Forest, XGBoost and Gradient Boosting with balanced class weights
- Save the best model to `wc2026_model.pkl`

### 2. Build the database

Run cell 5 to create `wc2026.db` with the World Cup 2026 group-stage fixtures.

### 3. Launch the web app

Run cell 9. Gradio will output a public URL:
```
Running on public URL: https://xxxxxx.gradio.live
```

### 4. Scoring system

| Event | Points |
|-------|:------:|
| Correct result (Win/Draw/Loss) | +3 |
| Exact score bonus | +2 |
| Wrong prediction | 0 |

## Model Performance

| Model | Accuracy | F1-macro |
|-------|:--------:|:--------:|
| Logistic Regression | ~52-55% | — |
| Random Forest | ~58-62% | — |
| **XGBoost / Gradient Boosting** | **~58-65%** | — |

> Note: predicting football results has an inherent accuracy ceiling of ~65-70% due to the sport's high variance. The model's value is in probability estimation, not point predictions.

### Top features by importance

1. `elo_diff` — Elo rating difference between teams
2. `diff_pts_5` / `diff_pts_10` — recent form differential
3. `home_elo` / `away_elo` — absolute team strength
4. `h2h_home_avg_pts` — historical head-to-head record
5. `home_win_streak` / `away_win_streak` — momentum

## Deploy to Hugging Face

1. Create a new Space at [huggingface.co/new-space](https://huggingface.co/new-space) — SDK: **Gradio**
2. Make sure your Space `README.md` header looks like:
   ```yaml
   ---
   title: WC2026 Match Predictor
   emoji: ⚽
   colorFrom: blue
   colorTo: green
   sdk: gradio
   app_file: app.py
   pinned: true
   ---
   ```
   (omit `sdk_version` to avoid version-mismatch errors)
3. Upload these files:
   - `app.py`
   - `requirements.txt`
   - `wc2026_model.pkl`
   - `wc2026_scaler.pkl`
   - `wc2026_features.pkl`
   - `wc2026.db`
4. Your app will be live at `https://huggingface.co/spaces/YOUR_USER/wc2026-match-predictor`
